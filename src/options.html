If images fail to download due to a permissions error, you can add the permission by entering the hostname from the failed URL and pressing the Request button. Your browser will then ask you to approve the new permission.

Valid hostnames contain only letters (a-z), numbers (0-9), and hyphens (-), in groups separated by a dot (.).

Added Host Permissions

NOTE: If the download request was redirected to a different host by the website server, both the original hostname and the redirect destination hostname may need permission added. The failure notice will only list the original hostname and finding the redirect destination can be tricky. One way to find the redirect hostname is to open the Network tab of the Web Developer tools for this page and repeat the download attempt, but you should only do that if you understand what you're doing.

Do not include the scheme, path, or port number with the hostname. The hostname may optionally be prefixed with "*." to also match any subdomain of a domain name. "*" by itself can also be used to match any hostname, but this is not recommended. For example, for the URL https://www.example.com:8080/folder/image.png, the hostnames www.example.com, *.www.example.com, *.example.com, and * will all match, but example.com will not.
